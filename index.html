Basic Data Structures

Array

A collection of elements stored at contiguous memory locations, accessible by index.

  • Insert: Adds a new element to the end of the array.
  • Delete: Removes the last element from the array.
  • Search: Finds an element in the array using linear search.
  • Sort: Sorts the array using the Bubble Sort algorithm.
  • Binary Search: Searches for an element in a sorted array using binary search.
  • Reset: Clears the array and removes all elements.
Time Complexity:
AccessO(1)
SearchO(n) - LinearO(log n) - Binary
Insert/DeleteO(n) - MiddleO(1) - End
String

A sequence of characters, typically used to represent text data.

  • Access: Accesses a character at a specific index.
  • Substring: Extracts a substring from the string.
  • Pattern Search: Searches for a pattern in the string using a pattern matching algorithm.
  • Reverse: Reverses the string.
Time Complexity:
AccessO(1)
SubstringO(n)
Pattern SearchO(n*m)
ReverseO(n)

Linear Data Structures

Linked List

A linear data structure where elements are stored in nodes, and each node points to the next node in the sequence.

  • Insert: Adds a new node to the beginning or end of the list.
  • Delete: Removes a node from the beginning or end of the list.
  • Traverse: Iterates through the list to access elements.
Time Complexity:
InsertionO(1) - BeginningO(n) - End
DeletionO(1) - BeginningO(n) - End
SearchO(n)
Stack

A LIFO (Last In, First Out) data structure where elements are added and removed from the top.

  • Push: Adds an element to the top of the stack.
  • Pop: Removes the element from the top of the stack.
  • Peek: Views the element at the top of the stack without removing it.
Time Complexity:
PushO(1)
PopO(1)
PeekO(1)
Queue

A FIFO (First In, First Out) data structure where elements are added at the rear and removed from the front.

  • Enqueue: Adds an element to the rear of the queue.
  • Dequeue: Removes an element from the front of the queue.
  • Front: Views the element at the front of the queue without removing it.
Time Complexity:
EnqueueO(1)
DequeueO(1)
FrontO(1)

Tree-based Data Structures

Binary Tree

A hierarchical data structure where each node has at most two children, referred to as left and right child.

  • Insertion: Adds a new node to the tree, maintaining the binary tree property.
  • Deletion: Removes a node from the tree, maintaining the binary tree property.
  • Search: Finds a node in the tree using a search algorithm.
Time Complexity:
InsertionO(log n) - Balanced
DeletionO(log n) - Balanced
SearchO(log n) - Balanced
Heap

A specialized tree-based data structure that satisfies the heap property (max-heap or min-heap).

  • Insertion: Adds a new element to the heap, maintaining the heap property.
  • Extract Max/Min: Removes the maximum or minimum element from the heap.
  • Heapify: Rebuilds the heap from an array.
Time Complexity:
InsertionO(log n)
Extract Max/MinO(log n)
HeapifyO(n)

Graph Data Structures

Graph

A non-linear data structure consisting of vertices (nodes) and edges that connect these vertices.

  • Add Node: Adds a new vertex to the graph.
  • Add Edge: Adds a new edge between two vertices.
  • DFS: Performs Depth-First Search on the graph.
  • BFS: Performs Breadth-First Search on the graph.
Time Complexity:
DFS/BFSO(V + E)
Add NodeO(1)
Add EdgeO(1)

Hash Tables

Hash Table

A data structure that implements an associative array abstract data type, mapping keys to values using a hash function.

  • Insertion: Inserts a key-value pair into the hash table.
  • Deletion: Deletes a key-value pair from the hash table.
  • Search: Searches for a key in the hash table.
  • Show Collision: Demonstrates how collisions are handled in the hash table.
Time Complexity:
InsertionO(1) - Average
DeletionO(1) - Average
SearchO(1) - Average

Trie (Prefix Tree)

Trie

A tree-like data structure used to store a dynamic set of strings, where each node represents a character.

  • Insert: Inserts a new word into the trie.
  • Search: Searches for a word in the trie.
  • Prefix Search: Searches for all words that start with a given prefix.
Time Complexity:
InsertionO(m) - m is word length
SearchO(m) - m is word length
Prefix SearchO(m) - m is prefix length
